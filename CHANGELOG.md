# 📝 更新日志 (Changelog)


## [v3.8.1] - 2026-03-21

### 🧩 工作区统一与状态稳定
- **统一 🧭 账号工作区语言**：重做账号侧边栏、详情区顶部状态区域与任务卡片，使侧栏、详情与任务列表统一为同一套 Pure Dark 控制台语言，多选模式下任务数始终可见。
- **修复 🪪 账号状态探活**：账号状态检测改为更稳定的独立 probe 路径，减少共享 client 带来的误判；已连接状态在主界面中显示为更明确的绿色成功态。
- **修复 📋 任务列表与计数异常**：修复 `sign task store` 全局缓存污染导致的任务数归零/列表为空问题，账号切换时任务区与状态区的体验更稳定。

### 🔁 任务反馈与运维同步
- **修复 ✅ 创建任务成功却提示失败**：创建/更新/删除任务在调度同步失败时不再整体返回失败，前端反馈改为“业务成功优先，调度同步仅记 warning”。
- **增强 📊 运维概览实时同步**：运维概览改用当前真实账号/任务数据计算摘要，并在主界面任务或账号变更后主动刷新，减少主界面与运维面板计数不一致的问题。

### 🚑 迁移热修复与发布流程
- **修复 🧬 daily-run 迁移识别**：补齐 `202603210001/202603210002` revision 识别与 schema-aware 对齐，避免 daily-run 相关 Alembic 版本被误判或反复重跑。
- **优化 🐳 镜像发布流程**：移除独立 CI workflow，保留 Docker Image workflow，并为 docs-only 改动增加忽略规则，减少无意义的镜像构建触发。


## [v3.7.5] - 2026-03-21

### 🚑 升级兼容与交互修复
- **修复 🧬 旧库升级兼容**：补齐 `alembic_version` 缺失与空表两类旧库状态的启动引导，旧版安装升级到 `v3.7.x` 时不再要求删库重建，已登录账号、TG Bot 通知、Telegram API 与全局设置可继续保留。
- **统一 🖥️ 主界面单入口交互**：任务工作台与旧子路由统一收回 `/dashboard`，旧的 `/dashboard/account-tasks`、`/dashboard/sign-tasks`、`/dashboard/sign-tasks/create` 改为兼容跳转壳，返回主界面时账号上下文不再丢失。
- **修复 🪪 账号状态与反馈体验**：兼容旧 `.session` 账号在 `TG_SESSION_MODE=string` 下的状态检测，补齐账号状态灯、版本号同步和稳定的 toast 自动消失逻辑。

### 📅 Daily Run MVP
- **新增 🗂️ 每日执行账本**：引入 `daily_task_runs` 表，为每个启用任务生成当日唯一执行记录，记录计划时间、状态、尝试次数、错误码和最近执行时间。
- **新增 🧠 分散式随机排程**：随机时间窗口任务不再各自独立随机，而是通过分槽 + 抖动算法在窗口内均匀铺开，显著降低任务扎堆到尾部的概率。
- **新增 🚚 Daily Dispatcher**：现已支持按 `planned_run_at` 扫描到期任务并拉起执行，`daily_run` 模式下默认接管签到任务调度，避免 legacy `sign-*` job 双跑。
- **新增 🔁 失败重试与截止补偿**：为 daily run 增加 `retry_wait`、`next_retry_at`、`deadline_at`、`max_attempts`，网络类/超时类错误会在当日截止前自动重试，超过截止时间则标记为 `expired`，会话失效类错误继续直接 `blocked`。
- **新增 📊 今日执行概览面板**：设置页运维概览新增“今日执行概览”，可查看今日总计划数、成功/待执行/重试中/阻塞/过期数量，以及最近 10 条 daily run 记录。


## [v3.7] - 2026-03-20

## [v3.5] - 2026-03-17

### ✨ 核心重构与交互升级
- **重构 💎 UI/UX 全面模态化**：由 Linear 风格驱动的自定义确认弹窗，彻底替换原生浏览器 `confirm()`。所有删除、重置、清空等敏感操作均拥有平滑的缩放动画与半透明磨砂背景。

- **增强 🎇 交互动效与反馈**：
    - 新装“立即终止 (Terminate)”任务流确认界面，强化操作安全感。
    - 优化了神经同步 (Neural Sync) 数据的导入反馈，增加进度感官暗示。
